✅ 宝塔面板访问限制设置教程

方式一：对 index.html 和 upload.php 设置访问密码

  1. 打开 宝塔面板，进入「网站」页面。
  2. 选择你想要设置的站点，点击进入。
  3. 找到「目录保护」功能。
  4. 添加需要限制访问的路径：
    • /index.html
    • /upload.php
  5. 为这两个路径设置账号和密码。
这样设置后，只有输入正确的账号密码才能访问上传页面和上传接口，rand-music.php 和 /music/ 等功能则不受影响。

方式二：更安全推荐（仅上传路径验证）

如果你更注重安全性，推荐将上传相关文件移动到一个新目录：

  1. 创建一个新目录，例如：/admin/
  2. 将 index.html 和 upload.php 移动到 /admin/ 目录中。
  3. 在宝塔面板中对 /admin/ 设置目录访问限制。

修改后，站点结构如下：

你的域名/
├── admin/
│    ├── index.html ← 上传页面
│    └── upload.php ← 上传逻辑
├── rand-music.php ← API接口（无需限制）
├── music/
└── music-list.json
推荐此结构用于长期部署，不仅安全性更高，而且结构更清晰。

🔐 小结

如果你希望：

  • ✅ 只保护上传功能（防止滥用）；
  • ✅ API 接口畅通无阻（如 rand-music.php 可供用户访问）；

那么强烈建议你：

  • 限制 /index.html 和 upload.php 的访问，
  • 或者将上传相关功能放入 /admin/ 目录并整体限制。

这样的设置结构清晰、安全性高，同时不会影响前端或外部平台正常调用 API 接口。
